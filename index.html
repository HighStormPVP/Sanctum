Agent options · this chat

Path allowlist

One path per line. When non-empty, file tools (read_file, list_dir, write_file, apply_patch) refuse paths outside these roots. Use ~ for your home directory. Leave empty for no restriction.

Pre-approved commands

One command (or prefix) per line. Matching shell commands run without an approval modal — useful for trusted read-only commands like git status. Anything else still requires approval when Approval Mode is on.

Step limit

Maximum number of tool-call rounds the agent can run per turn. Higher = lets it complete complex tasks; lower = caps runaway loops. Default 5.

Context window

How many tokens of conversation the model can remember. Bigger uses more memory. The KV cache for a 32K window on Qwen3 30B-A3B alone eats ~13 GB.

4096 — tightest, ~10 GB total for Qwen3
8192 — safe default for 32 GB systems
16384 — good for ~48 GB systems
32768 — Qwen3's native max, needs ~36 GB

API Keys (cloud models)

Sanctum is local-first, but you can also call cloud models. Keys are stored in this app's local settings file on disk — no other process sees them. Picks appear under Cloud Models in the model picker. Leave blank to keep cloud picks disabled.

Get a key at console.anthropic.com. Pricing per million tokens varies by model — Haiku is cheapest, Opus is the most expensive.
Get a key at aistudio.google.com. Gemini 2.5 Flash has a generous free tier; Pro is paid per million tokens.

MCP Servers

Model Context Protocol servers expose external tools (Blender, Playwright, etc.) to agentic chats. Each entry runs as a local subprocess; its tools appear alongside Sanctum's built-ins whenever an agentic model is active.

MCP Servers — JSON Config

Same shape as Claude Desktop and Cursor — paste a config block under mcpServers. Each entry needs a command, optional args array, and optional env object.
